BSA/AML Transaction Risk Detection

A lightweight, interpretable compliance analytics project that combines unsupervised anomaly detection with BSA/AML typology rules to create a prioritized suspicious activity review queue.

What the project does

The pipeline generates synthetic transactions, engineers BSA-relevant features, scores unusual activity with Isolation Forest, applies typology rules, and exports a high/critical-risk queue for review.

ML + RulesCombined detection approach
0–100Composite risk score
4Embedded AML typologies

Typologies covered

  • Structuring / smurfing through repeated near-threshold deposits
  • Rapid movement / layering through high-value wire activity
  • Dormant account spike activity after long inactivity gaps
  • Round-dollar anomaly patterns in higher-value transactions

How to run it

Install dependencies and run the demo:

pip install -r requirements.txt

python demo.py

The run exports flagged_transactions.csv, sensitivity_analysis.csv, and aml_visuals.png.

Compliance positioning

This is a portfolio demonstration, not a production SAR decisioning system. The value is in showing how financial-crimes domain knowledge, model-risk thinking, reproducible Python, and explainable rules can work together in a defensible analytical workflow.
